# AZT — MASTER INDEX

Дата: 2026-04-13

## Назначение

Это главная точка входа по проекту AZT.  
Документ нужен, чтобы быстро понять:

- какие материалы уже подготовлены
- какой документ для чего нужен
- в каком порядке ими пользоваться

---

## 1. Базовые документы

### 1. Полный аудит

Файл:

- `AZT — Полный аудит 2026-04-12-codex.md`

Назначение:

- полный разбор сайта
- SEO, GEO, контент, техника
- постраничные проблемы
- список битых URL и системных ошибок

Когда использовать:

- если нужно понять текущее состояние проекта
- если нужно обосновать, почему требуется работа

### 2. Fix plan

Файл:

- `AZT — Fix plan для разработчика 2026-04-13-codex.md`

Назначение:

- фиксирует порядок исправлений
- отвечает на вопрос "что чинить сначала"

Когда использовать:

- перед постановкой задач в работу
- для приоритизации техблока

### 3. ТЗ для разработчика Bitrix

Файл:

- `AZT — ТЗ для разработчика Bitrix 2026-04-13-codex.md`

Назначение:

- формализует задачи для разработки
- содержит критерии приёмки

Когда использовать:

- при передаче задач Bitrix-разработчику
- при контроле выполнения

## 2. Документы по стратегии и развитию

### 4. SEO/GEO roadmap

Файл:

- `AZT — SEO-GEO roadmap 30-60-90 2026-04-13-codex.md`

Назначение:

- показывает очередность действий на 30/60/90 дней
- связывает техработы, SEO, контент и GEO

Когда использовать:

- для планирования проекта по этапам
- для синхронизации команды

### 5. Контент-план

Файл:

- `AZT — Контент-план SEO-GEO 2026 Q2-Q3-codex.md`

Назначение:

- описывает, какие страницы усиливать
- какие новые материалы создавать
- как строить контент под Яндекс

Когда использовать:

- для SEO-специалиста
- для редактора/контентщика
- для планирования публикаций

## 3. Документы для бизнеса и сервисов Яндекса

### 6. Exec summary

Файл:

- `AZT — Exec summary для собственника 2026-04-13-codex.md`

Назначение:

Файл:

- `AZT — План работ по Яндекс Вебмастеру и Яндекс Бизнесу 2026-04-13-codex.md`

- после техфиксов

## 6. Что делать дальше

Проект готов к переходу из стадии анализа в стадию исполнения.

Логика следующая:

1. согласовать приоритеты с собственником
2. отдать техблок в работу
3. после техфиксов обновить Яндекс Вебмастер и Яндекс Бизнес
4. затем запускать усиление контента и money pages
