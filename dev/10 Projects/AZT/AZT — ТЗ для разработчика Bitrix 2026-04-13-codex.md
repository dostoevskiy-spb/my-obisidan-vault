# AZT — ТЗ для разработчика Bitrix

Дата: 2026-04-13  
Основа: полный аудит, fix plan, roadmap

## Назначение документа

Это ТЗ под реализацию в Bitrix.  
Формат: задача -> что сделать -> ожидаемый результат -> критерий приёмки.

Цель:

- убрать критические SEO-ошибки
- стабилизировать индексацию
- починить региональность
- подготовить сайт под дальнейшее SEO/GEO-усиление

---

## Блок 1. Индексация и sitemap

### Задача 1. Пересобрать sitemap по реальным URL

Что сделать:

- проверить генерацию всех sitemap в Bitrix
- исключить из sitemap URL со статусами `404`, `410`, `500`
- исключить служебные страницы:
  - `/404.php`
  - `/thank.php`
  - аналогичные служебные URL, если есть
- включить в sitemap только реальные индексируемые страницы со статусом `200`
- добавить в sitemap страницы, которые сейчас живые, но отсутствуют:
  - `/contacts/`
  - `/delivery/`
  - `/portfolio/`
  - `/reviews/`
  - `/catalog/polikarbonat/`
  - `/catalog/gryadki/`
  - `/actions/`
  - `/policy/`
  - `/privacy/`

Отдельно:

- в `sitemap-iblock-9.xml` заменить статьи без `/news/` на реальные URL с `/news/`

Ожидаемый результат:

- все URL из sitemap открываются с `200`
- ни один URL из sitemap не ведёт на ошибку
- sitemap отражает фактическую структуру сайта

Критерий приёмки:

- выборочная ручная проверка 20 URL из каждого sitemap
- полная автоматическая проверка статусов `200` по всем URL карты

### Задача 2. Нормализовать индексируемый набор страниц

Что сделать:

- сформировать перечень страниц, которые можно индексировать
- для служебных, thank, success, 404 и подобных — выставить `noindex`
- убедиться, что такие страницы не попадают в sitemap

Критерий приёмки:

- в индексационный набор не попадают служебные страницы

## Блок 2. 404 и служебные шаблоны

### Задача 3. Исправить шаблон 404

Что сделать:

- сохранить корректный HTTP-статус `404`
- для 404-шаблона установить:
  - корректный `title`
  - корректный `description` или отсутствие, если так лучше
  - `meta robots="noindex, follow"`
- убрать мусорные старые мета-данные, не относящиеся к тематике сайта
- убрать невалидный или вредный canonical с 404-страницы

Ожидаемый результат:

- 404 не индексируется
- 404 не выглядит как обычная контентная страница

Критерий приёмки:

- любая несуществующая страница отдаёт `404`
- в HTML есть `noindex`
- отсутствуют мусорные meta

### Задача 4. Закрыть thank/success страницы

Что сделать:

- найти все URL подтверждения отправки форм
- выставить на них `noindex, nofollow` или как минимум `noindex`
- исключить их из sitemap

Критерий приёмки:

- `thank`-страницы не индексируются и не присутствуют в sitemap

## Блок 3. Внутренняя перелинковка

### Задача 5. Исправить битые и кривые внутренние ссылки

Проблемы, подтвержденные аудитом:

- ссылки на статьи без `/news/`
- ссылки вида `/prm/catalog/catalog/...`

Что сделать:

- найти шаблоны и компоненты, где генерируются эти ссылки
- исправить логику построения URL
- перепроверить:
  - главную
  - блок статей
  - блок хитов и категорий
  - региональные страницы
  - футер
  - breadcrumbs

Ожидаемый результат:

- сайт не генерирует битые URL
- сайт не генерирует лишние префиксы и дубли сегментов

Критерий приёмки:

- сканирование HTML главной и ключевых разделов не показывает мусорных ссылок

## Блок 4. Canonical и региональность

### Задача 6. Исправить canonical

Что сделать:

- проверить шаблоны canonical на всех типах страниц
- убрать двойной слеш на главной
- canonical должен вести только на существующую страницу
- исключить canonical на ошибки и служебные URL

Критерий приёмки:

- canonical у главной ровно `https://azt-teplica.ru/`
- canonical не содержит двойных слешей и не ведёт на 404

### Задача 7. Привести региональные страницы к одной логике

Что сделать:

- согласовать с SEO стратегию:
  - отдельные индексируемые региональные страницы
  - или одна каноническая версия

Если регионы индексируемые:

- сделать уникальные canonical
- сделать `hreflang`
- уникализировать title/description/H1 по городу

Если регионы не индексируемые:

- выстроить последовательную канонизацию/редиректы
- убрать противоречащие sitemap

Критерий приёмки:

- региональные страницы не конфликтуют между собой
- поисковик получает один понятный сигнал по каждому URL

### Задача 8. Добавить `hreflang`, если выбираем индексируемую мультирегиональность

Что сделать:

- проставить `hreflang` между версиями:
  - Пермь
  - Екатеринбург
  - Тюмень
  - Ижевск
- проверить соответствие canonical и `hreflang`

Критерий приёмки:

- на всех региональных аналогах есть перекрёстные `hreflang`

## Блок 5. Meta и Open Graph

### Задача 9. Нормализовать генерацию title и description

Что сделать:

- привести шаблоны title/description к единым правилам
- убрать явные артефакты:
  - `в Перми в Перми`
  - неестественные формулировки
  - избыточные хвосты
- сократить слишком длинные шаблонные title

Критерий приёмки:

- на ключевых шаблонах title и description читаемы и не содержат технических артефактов

### Задача 10. Убрать мусорный meta layer

Что сделать:

- удалить `meta keywords="."`
- проверить отсутствие дублирующихся мета-тегов
- исправить HTML entities в мета-данных

Критерий приёмки:

- в HTML нет `keywords="."`
- в OG и title не протекают `&quot;` и подобные сущности

### Задача 11. Исправить Open Graph

Что сделать:

- на главной исправить `og:title`
- на `/about/` исправить `og:title`
- на `/delivery/` убрать `og:title = Title`
- на статьях подставлять название статьи
- по возможности подготовить разные `og:image` по шаблонам

Критерий приёмки:

- OG соответствует содержанию страницы

## Блок 6. Schema.org

### Задача 12. Добавить schema для компании

Что сделать:

- добавить `Organization`
- при необходимости `LocalBusiness`
- заполнить:
  - название
  - URL
  - логотип
  - телефоны
  - email
  - адрес
  - регион

Критерий приёмки:

- главная и/или контакты содержат валидную schema организации

### Задача 13. Усилить `Product`

Что сделать:

- на карточках товаров добавить:
  - `description`
  - `image`
  - `brand`
  - `offers.price`
  - `availability`
  - `url`
  - при наличии `sku`

Критерий приёмки:

- карточка товара проходит валидацию по Product schema без критичных ошибок

### Задача 14. Добавить `Article` / `BlogPosting`

Что сделать:

- на статьях вывести schema с:
  - `headline`
  - `description`
  - `author`
  - `datePublished`
  - `dateModified`
  - `image`
  - `mainEntityOfPage`

Критерий приёмки:

- реальные статьи в `/news/...` содержат корректную schema article

### Задача 15. Добавить `BreadcrumbList`

Что сделать:

- разметить хлебные крошки на категориях, карточках и статьях

Критерий приёмки:

- breadcrumbs доступны в HTML и размечены schema

  - комплектация

  - гарантия
  - доп. опции

Критерий приёмки:

## Блок 8. Производительность

### Задача 18. Разобраться с отключенным кэшированием публичных страниц

Что сделать:

- найти причину заголовков:
  - `cache-control: no-store, no-cache, must-revalidate`
  - `pragma: no-cache`
- проверить:
  - настройки Bitrix
  - композитный режим
  - прокси/сервер
  - кастомные заголовки в шаблонах и модулях

Критерий приёмки:

- публичные страницы не запрещают кэширование без причины

### Задача 19. Lazy loading и медиа

Что сделать:

- проставить `loading="lazy"` на изображения ниже первого экрана
- проверить, что не ломается CLS и не ухудшается UX

Критерий приёмки:

- у второстепенных изображений есть lazy loading

### Задача 20. Уменьшить лишние фронтовые зависимости

Что сделать:

- проверить общие подключения JS/CSS на шаблонах
- убрать модули Bitrix, которые не нужны на каждой странице

Критерий приёмки:

- на типовых шаблонах нет очевидного фронтового мусора
