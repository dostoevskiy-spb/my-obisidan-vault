## 4. Критические баги

### 4.1 Title на региональных страницах — "Пермь" вместо города

| Регион | URL | Title содержит | H1 содержит | Должно быть в title |
|--------|-----|----------------|-------------|---------------------|
| Екатеринбург | `/ekb/` | "в Перми" | "в Екатеринбурге" | "в Екатеринбурге" |
| Тюмень | `/tmn/` | "в Перми" | "в Тюмени" | "в Тюмени" |
| Ижевск | `/izh/` | "в Перми" | "в Ижевске" | "в Ижевске" |

**H1 корректный, title — нет. Региональное SEO для 3 городов полностью сломано.**

### 4.2 Битые URL в sitemap

Sitemap содержит URL, возвращающие 404:
- `/404.php` — страница 404 сама в sitemap
- `/promyshlennaya-teplitsa/` — 404
- `/actions/rassrochka/` — 404
- `/actions/khranenie/` — 404
- `/actions/dostavka/` — 404
- Корневые URL статей (вне `/news/`) — 404

### 4.3 Дубли контента без canonical

Каждая страница существует в 4+ вариантах без rel=canonical:
- `/catalog/teplicy/` = `/prm/catalog/teplicy/` = `/ekb/catalog/teplicy/` = ...
- `/about/` = `/prm/about/` = ...
- и т.д.

### 4.4 H1 "Стеклянные в Перми"

На `/catalog/teplicy/steklyannye/` — потеряно слово "теплицы" в H1. Шаблонный баг.

15. **robots.txt — директива `Host:`** — устаревшая, но оставить для совместимости. Проверить, что каноничный домен указан корректно.

### 10.3 Что менее приоритетно для Яндекса

- **Open Graph** — важен для шеринга, но не влияет на выдачу Яндекса напрямую. Можно делать во вторую очередь.
- **hreflang** — Яндекс понимает его хуже Google. Основной инструмент — **региональная привязка в Вебмастере**.
- **llms.txt** — Яндекс пока не декларировал поддержку (в отличие от OpenAI/Anthropic). Делать в последнюю очередь, но YandexGPT может подтянуть косвенно.
- **Google Analytics** — можно не приоритезировать, если Google-трафик не в фокусе.
- **Google Search Console** — аналогично.

### 10.4 GEO для YandexGPT / Нейро

Яндекс активно развивает Нейро (поиск с ИИ):
- **Структурированные данные** — YandexGPT подтягивает из Schema.org
- **Чёткие ответы на вопросы** — FAQ-блоки критичны
- **Факты и цифры с контекстом** — сколько производим, гарантия, регионы
- **Entity-разметка компании** — LocalBusiness с полными данными
- **Блог с экспертным тоном** — Нейро цитирует источники

## 11. Приоритеты исправлений (с фокусом на Яндекс)

### КРИТИЧЕСКИЕ (сделать в первые 1–2 недели)

1. **Яндекс.Вебмастер** — подключить, загрузить sitemap, настроить региональную привязку для всех 4 городов (самое быстрое действие с большим эффектом)
2. **Яндекс.Бизнес / Справочник** — создать карточку для каждого города (Пермь, Екатеринбург, Тюмень, Ижевск). Это даст буст в локальной выдаче Яндекса
3. **Title на региональных страницах** — "Пермь" вместо правильного города (3 региона сломаны) — критично для регионального ранжирования в Яндексе
4. **Юридические страницы** — `/privacy/` содержит `https://ваш-сайт.ru`, `/policy/` содержит "Образец" — Яндекс банит за такой "сайт-шаблон" через алгоритм "Минусинск" и снижает ИКС
5. **Копипаст между категориями** — убрать тепличные блоки со страниц беседок, грядок, поликарбоната, парников. Яндекс особенно жёсток к неуникальности (алгоритм "Баден-Баден")
6. **Шаблонный мусор** в HTML — "Загрузка данных товара...", комментарии к статьям на карточках. Ухудшает поведенческие и ИКС
7. **Meta description** на все страницы (90%+ без description) — напрямую влияет на CTR в выдаче Яндекса
8. **Битые URL** — `/foundation`, `/bochka`, `/promyshlennaya-teplitsa/`, `/actions/rassrochka/` — удалить через Вебмастер ("Удаление URL"), вычистить sitemap
9. **Canonical URL** на всех страницах — для устранения дублей (`/prm/`, `/ekb/` и т.д.)

### ВЫСОКИЕ (2–6 недель)

10. **Schema LocalBusiness** для каждого города — Яндекс использует для расширенных сниппетов
11. **Schema Review + AggregateRating** — Яндекс показывает звёзды в выдаче (большой буст CTR)
12. **Schema FAQPage** на каталогах и статьях — Яндекс активно использует для расширенных сниппетов
13. **Product Schema** полная (цена, описание, наличие, brand) — для товарной выдачи Яндекса
14. **BreadcrumbList Schema** — Яндекс показывает в сниппете
15. **Стимулирование отзывов на Яндекс.Картах** для каждого города — критично для поведенческих и локал-SEO
16. **Яндекс.Метрика — настроить цели** (заявка, звонок, добавление в корзину), Вебвизор, Карта кликов
17. **Страница "Доставка"** — переписать под реальную логистику теплиц (зоны, сроки по сезонам, стоимость, FAQ по доставке)
18. **H1 "Стеклянные в Перми"** — исправить (потеряно слово "теплицы")

22. **Скорость загрузки** — Яндекс очень чувствителен к скорости (бандлинг CSS/JS, lazy loading). Проверить через Яндекс.Метрику → Отчёты → Скорость загрузки

### СРЕДНИЕ (1–3 месяца)

23. **Турбо-страницы** для блога — буст в мобильной выдаче Яндекса
24. **Яндекс.Маркет** — листинг готовых теплиц и комплектующих (доп. канал + отзывы для ранжирования)
