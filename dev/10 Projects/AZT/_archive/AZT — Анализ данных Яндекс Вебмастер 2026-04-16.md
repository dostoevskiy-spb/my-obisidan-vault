## 1. Назначение документа

Этот документ сверяет данные Яндекс.Вебмастера с тезисами итогового отчёта для собственника.

Задачи:

- согласовать численные данные из Вебмастера с выводами итогового отчёта;
- указать, какие тезисы подтверждаются данными Вебмастера;
- указать, какие тезисы требуют уточнения;
- отделить данные Вебмастера от фактов, которые были подтверждены живой проверкой сайта, но не проверяются через Вебмастер;
- зафиксировать новые данные, которые появились после выгрузки из Вебмастера.

Документ не содержит управленческих оценок. Формулировки ниже построены как: факт → источник → статус → следующее действие.

## 4. Согласование цифр sitemap

В итоговом отчёте по живой проверке указано:

- `57` URL в 5 основных sitemap;
- `56` уникальных URL;
- `14` URL отдают `404`;
- среди проблемных URL есть `404.php`, `thank.php`, неверные URL статей и кейсов.

В Вебмастере указано:

- один индексный sitemap `https://azt-teplica.ru/sitemap.xml`;
- `children_count = 20`;
- `urls_count = 20`;
- `errors_count = 0`.

Эти данные не являются прямым противоречием.

Причина:

- живая проверка считала URL внутри выбранных sitemap-файлов и проверяла HTTP-ответ каждого URL;
- Вебмастер в поле `errors_count` отражает наличие ошибок обработки sitemap, а не полную HTTP-проверку всех URL как отдельный аудит;
- Вебмастер подтверждает наличие URL-проблем через другие отчёты: `HTTP_ERROR`, `broken_internal_links_samples`, `search_events_samples`.

Итоговая согласованная формулировка:

> Вебмастер принимает `sitemap.xml` как валидный индексный sitemap без XML-ошибок, но другие отчёты Вебмастера и живая проверка сайта подтверждают наличие ошибочных URL, редиректов и неканонических страниц. Поэтому задача пересборки sitemap остаётся актуальной, но в отчёте нужно уточнять, что проблема не в XML-валидности sitemap, а в составе URL и связанной URL-структуре сайта.

Сохраняются как подтверждённые живой проверкой сайта:

- юридические страницы с заглушками;

- отсутствие/неполнота schema на проверенных шаблонах;
- нерелевантный FAQ на категориях;
- cache-control `no-store, no-cache, must-revalidate`;
- отсутствие lazy loading на коммерческих страницах;
- OG/meta-проблемы, найденные в HTML.

Причина: Вебмастер не отдаёт эти HTML-фрагменты в данной выгрузке, поэтому он не может быть источником подтверждения или опровержения по этим пунктам.
