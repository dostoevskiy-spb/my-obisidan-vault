## Что не удалось получить

1. **Настройки 2 МК-кампаний** (`709149630`, `709150108`) — `campaigns.get` возвращает пустой массив для всех типов, хотя кампании есть в отчётах. Вероятно, Мастер Кампаний в API v5 обрабатывается отдельным сервисом (`smart_ad_targets.get` или специфическим endpoint). Текущий токен не даёт к нему доступа или токен-агентский.
2. **Общий набор минус-слов** — `direct_negative_keywords_get` требует IDs; попытка без IDs вернула ошибку. Вероятно, общих наборов в аккаунте нет (подтверждается `NegativeKeywordSharedSetIds: []` на всех проверенных кампаниях).
3. **Баланс рекламного счёта** — API v5 не отдаёт его (`balance.yandex.ru` отдельно). Косвенно видно через `AccountQuality`, `Restrictions` и `Spend`.

## Как использовать дальше

Выгрузка — фактологическая база для:

1. Финальной сверки рассказа агентства про «3 слоя рекламы» с реальным статусом кампаний в кабинете. Результат: 4 новых ЕПК + 2 МК активны, 4 старых SUSPENDED.
2. Проверки стратегий автооптимизации на корректность выбранной цели (`GoalId`). Ключевой вопрос: почему оптимизация идёт на `547431898` (`thank.php`, 6 срабатываний/месяц), а не на `13` (Вовлечённые визиты, сотни срабатываний).
3. Пересборки блока «Пул посадочных» для МК/ЕПК — вытащить список товарных URL через `direct_ads_get` + `CRITERIA_PERFORMANCE_REPORT` для проверки, какие ключи ведут на какие страницы.
4. Инвентаризации ключевых фраз и минус-слов — 278 ключей поштучно, плюс встроенные минусы. Сверить с планом агентства из [[AZT — Входное семантическое ядро агентства 2026-04-17-cx]].
5. Финансового план-факта за пик сезона — `CAMPAIGN_PERFORMANCE_REPORT` по дням + прогноз оставшихся 43 дней пика.
6. Формирования вопросов агентству в партнёрском тоне — по конкретным фактам из выгрузки, без домыслов.
