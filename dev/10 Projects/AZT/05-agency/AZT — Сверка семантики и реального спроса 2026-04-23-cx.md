## 8. Методологическое замечание — 4 типа соотношений Wordstat ↔ Direct

Соотношение теории и факта неравномерно по кластерам:

### Тип А: «высокий захват» (Direct ≈ 100% Wordstat-теории)
Наше ядро покрывает кластер, реклама берёт почти весь доступный спрос.
**Пример:** «купить теплицу» (Wordstat `30 570/мес`, Direct `~32 000/мес`).
**Рекомендация:** масштабировать бюджет и позиции.

### Тип Б: «узкая фраза ≠ широкий кластер» (Direct > Wordstat по узкой формулировке)
Wordstat мало по одной фразе, но ядро собрано из синонимов — в Directе виден весь кластер.
**Пример:** каплевидные (Wordstat узкое `194`, Direct экстраполяция `~1 100/мес`).
Происходит потому что люди ищут один продукт разными словами:

| Формулировка | Wordstat РФ/мес |
|---|---:|
| `теплица каплевидная` | `194` |
| `теплица капелька` | `2 697` (×14) |
| `капельная теплица` | `1 433` (×7) |
| `теплица капля` | `432` (×2) |
| **Всего по кластеру «капля/капелька/каплевидная»** | **`~4 756`** (×24 к узкой формулировке) |

**Методологический урок:** для оценки потенциала кластера нельзя брать одну фразу из Wordstat. Надо собрать **все лексические варианты продукта**, просуммировать их частотности, добавить гео/уточнения.

### Тип В: «Wordstat есть, ядра нет» (Direct `~10-20%` Wordstat)
Теоретический спрос существенный, наше ядро узкое — большая часть проходит мимо.
**Примеры:**
- арочные (Wordstat `894`, Direct `~150/мес` = `17%` захвата)
- прямостенные (`429` vs `47/мес` = `11%`)
- под ключ (`558` vs `~40/мес` = `~7%`)

**Рекомендация:** **расширить ядро**, собрать варианты с размерами/гео/действиями.

### Тип Г: «слабо везде»
Спрос маленький и в теории, и на факте.
**Пример:** стеклянные в наших 4 регионах (`~565/мес` всего по Wordstat, Direct `63 показа / 7 дней`).
**Рекомендация:** не стоит вкладываться в масштабирование, можно поддерживать точечно (что агентство сейчас и делает через `SUSPENDED`).

**Практический вывод:** для адекватного анализа кластера нужно **оба источника** — Wordstat для понимания потолка и Direct для понимания захвата. По одному источнику можно неверно оценить возможности.

## 9. Комментарий к подходу агентства «опираемся на фактически собранную статистику»

Агентство в письме указало: *«наш подход основан на анализе фактически собранной статистики. Решения принимаем не только на основе прогнозных данных, а исходя из реального поведения пользователей, качества трафика и полученных конверсий.»*

**Методологически тезис правильный** — соглашаемся. Факт важнее прогноза, решения по CR и качеству трафика лучше, чем по Wordstat.

### Однако у подхода «только факт» есть слепые пятна

1. **Замкнутый цикл ядра.** Факт собирается только по ключам, которые уже в ядре. Если кластера нет в ядре — факта по нему не будет, и по «методу фактов» кластер «не работает». На деле это просто **не захвачено**.
   - Арочные: в Directе `35` показов за 7 дней при Wordstat `894/мес` — это не «арочные плохие», это «наше ядро арочные почти не ловит»
2. **Малая выборка новых ЕПК.** За 7 дней ЕПК дали `5-10` кликов на узкий кластер. Делать вывод «стеклянные не работают» на `0` конверсий при `5` кликах — статистически нельзя
3. **Качество трафика зависит от посадочной.** Все кластеры ведут на 2 URL. Высокие отказы на стеклянных могут быть не про ключи, а про то, что landing не про стеклянные
4. **Сезонность искажает короткую выборку.** Апрель-май — пик, июнь-июль — обвал в `2×`. Одна неделя не даёт полной картины
5. **Конкуренция между нашими кампаниями.** Если по одному запросу могут показаться `700165888`, `709131003` и `709149630` — «факт по кампании» искажён внутренним аукционом

### Конструктивное дополнение

Факт + теория = полная картина:
- **Факт (Direct) показывает**, что мы захватываем и с каким результатом
- **Теория (Wordstat/Webmaster)** показывает **мёртвые зоны** — где ядро узкое, и мимо нас проходит спрос
- **Webmaster `popular_queries`** покажет **органический факт** — по каким запросам сайт реально видят в выдаче Яндекса (включая те, по которым мы не в рекламе)

### Общее мнение по решению агентства о стеклянных

Их решение — **правильное по методологии**: собрали гипотезу из Wordstat (`565/мес` в 4 регионах, `238/мес` активируется), подготовили в `SUSPENDED`, планируют активировать и собирать факт. Это ровно цикл «прогноз → факт → решение».

**Вопрос только в явности критериев:** через сколько недель и по каким показателям принимается решение о дальнейшей судьбе кластера? При прогнозе `2-12 конверсий/мес` (CR 1-5% поиска) для значимой оценки нужно 6-10 недель. Предложим согласовать это явно на встрече.
