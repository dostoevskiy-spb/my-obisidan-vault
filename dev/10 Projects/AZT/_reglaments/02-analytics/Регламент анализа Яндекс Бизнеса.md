# AZT — Регламент анализа Яндекс Бизнеса

## Назначение

Яндекс Бизнес анализируется как отдельный канал привлечения, а не как вспомогательный раздел Метрики.

Карточки организаций могут давать показы, переходы на сайт, звонки, маршруты и другие действия без посещения сайта. Поэтому их нельзя оценивать только через обычный сайтовый счетчик.

## Источники

Для анализа используются автоматические счетчики организаций в Метрике.

Признаки таких счетчиков:

- в наших данных основной признак — `source = sprav`;
- сайт счетчика обычно `yandex.ru/maps`;
- название содержит организацию и адрес;
- права чаще всего только на просмотр;
- присутствуют цели карточки организации и/или `vacuum`-события;
- поведение посетителей сайта на них не анализируется.

## Не смешивать с сайтом

Счетчик сайта отвечает на вопрос:

```text
Что пользователи делают на azt-teplica.ru?
```

Счетчики Яндекс Бизнеса отвечают на вопрос:

```text
Что пользователи делают с карточками организации в сервисах Яндекса?
```

Эти данные нужно связывать, но не смешивать.

Для автоматических счетчиков организаций нельзя делать выводы по обычным поведенческим метрикам сайта:

- показатель отказов;
- глубина просмотра;
- время на сайте;
- ecommerce;
- поведение на URL сайта.

Эти метрики могут присутствовать в API-ответах, но для счетчиков Яндекс Бизнеса они не являются основанием для управленческих выводов.

## Обязательные срезы

Для каждого счетчика организации нужно выгружать:

- `поверхность → событие`;
- `поверхность → организация → событие`;
- `день → событие`;
- `день → поверхность`;
- `поисковая фраза → поверхность → событие`;
- `поисковая фраза → организация → событие`;
- цели счетчика организации;
- переходы на сайт;
- звонки;
- маршруты;
- показы организации;
- показы профиля.

## Ключевые метрики

Основные метрики отчета `Целевые события на Яндексе`:

- `ym:s:vacuumevents` — количество событий;
- `ym:s:visits` — визиты;
- `ym:s:vacuumusers` — пользователи;
- `ym:s:vacuumeventsPerUser` — событий на пользователя.

Основные измерения:

- `ym:s:vacuumSurface` — поверхность Яндекса;
- `ym:s:vacuumOrganization` — организация или адрес;
- `ym:s:vacuumEvent` — действие;
- `ym:s:searchPhrase` — поисковая фраза;
- `ym:s:date` — дата.

## Рецепты запросов

Все запросы выполняются по счетчику организации, с `accuracy=full` и желательно `lang=ru`.

Сводка по действиям:

```text
dimensions=ym:s:vacuumEvent
metrics=ym:s:vacuumevents,ym:s:visits,ym:s:vacuumusers,ym:s:vacuumeventsPerUser
sort=-ym:s:vacuumevents
```

Сводка по поверхностям:

```text
dimensions=ym:s:vacuumSurface
metrics=ym:s:vacuumevents,ym:s:visits,ym:s:vacuumusers,ym:s:vacuumeventsPerUser
sort=-ym:s:vacuumevents
```

Поверхность → организация → действие:

```text
dimensions=ym:s:vacuumSurface,ym:s:vacuumOrganization,ym:s:vacuumEvent
metrics=ym:s:vacuumevents,ym:s:visits,ym:s:vacuumusers,ym:s:vacuumeventsPerUser
sort=-ym:s:vacuumevents
```

Динамика действий:

```text
dimensions=ym:s:date,ym:s:vacuumEvent
metrics=ym:s:vacuumevents,ym:s:visits,ym:s:vacuumusers,ym:s:vacuumeventsPerUser
sort=ym:s:date
```

Поисковые фразы → поверхность → действие:

```text
dimensions=ym:s:searchPhrase,ym:s:vacuumSurface,ym:s:vacuumEvent
metrics=ym:s:vacuumevents,ym:s:visits,ym:s:vacuumusers
sort=-ym:s:vacuumevents
```

## Коммерческие действия

Коммерчески значимые действия не равны заявкам или продажам. Они показывают намерение пользователя и место карточки в воронке.

К коммерчески значимым действиям относятся:

- переход на сайт;
- клик на звонок;
- телефонный звонок;
- построить маршрут;
- движение по маршруту;
- переход в мессенджер;
- отправка заявки, если доступна;
- посещение организации, если доступно и интерпретируемо.

Показы организации и профиля являются верхним уровнем воронки. Это не лиды, но они показывают наличие спроса.

## Региональная связь

Для каждой карточки нужно сопоставлять:

- город и адрес карточки;
- региональную посадочную страницу сайта;
- состояние этой страницы в Вебмастере;
- переходы с карточки на сайт;
- поисковые фразы карточки;
- звонки и маршруты;
- рекламный трафик в этот регион, если доступен.

Если карточка дает спрос, а региональная страница имеет проблемы в Вебмастере, это усиливает приоритет региональной задачи.

## Примеры выводов

Корректно:

```text
Пермская карточка Яндекс Бизнеса показывает значимый объем событий. При этом региональная часть сайта имеет сигналы LOW_QUALITY в Вебмастере. Нужно проверить связку карточка → региональная посадочная → индексирование.
```

Некорректно:

```text
Карточка хорошо работает, значит с регионом все нормально.
```

Карточка и сайт являются разными каналами. Хорошая карточка не отменяет технических проблем сайта.
