# Регламент конкурентного анализа

## Назначение

Конкурентный анализ нужен не для общего описания конкурентов, а для поиска конкретных решений, которые помогут проект усилить онлайн-присутствие, посадочные страницы, карточки Яндекс Бизнеса, рекламу, SEO/GEO и коммерческие офферы.

Итог конкурентного анализа — не список "у кого красивее сайт", а карта:

```text
какой спрос есть
→ кто его забирает
→ за счет каких страниц, карточек, офферов и объявлений
→ что из этого применимо для проекта
→ что нужно сделать на сайте, в карточках, рекламе или контенте
```

## Место в общей модели

Конкуренты анализируются после базовой проверки собственных данных проекта:

1. Вебмастер показывает техническое состояние сайта.
2. Метрика показывает фактический трафик и цели.
3. Яндекс Бизнес показывает действия в карточках.
4. Wordstat показывает потенциальный спрос.
5. Конкуренты показывают, кто уже забирает этот спрос и за счет чего.

Если собственная техническая база проект не проверена, конкурентный анализ может привести к неверному выводу: например, "нужно писать статьи", хотя сначала нужно исправить индексирование или посадочные.

## Источники

Для конкурентного анализа используются:

- поисковая выдача Яндекса;
- Яндекс Карты;
- сайты конкурентов;
- карточки конкурентов в Яндекс Бизнесе;
- рекламные объявления, если видны;
- Wordstat;
- Вебмастер и Метрика проекта;
- медиаплан и семантика агентства;
- отзывы, рейтинги, фото, товары, акции, ответы в карточках.

## Сегменты анализа

Конкуренты сравниваются по сегментам:

- коммерческие запросы;
- региональные запросы;
- брендовые запросы;
- запросы с ценой;
- запросы с отзывами;
- запросы по доставке и монтажу;
- запросы по усиленным теплицам;
- информационные запросы;
- карточки Яндекс Карт;
- рекламные офферы;
- посадочные страницы;
- SEO-статьи.

## Что фиксировать по каждому конкуренту

Минимальная карточка конкурента:

- название;
- регион;
- сайт;
- карточка Яндекс Бизнеса;
- запросы, по которым найден;
- тип видимости: SEO, Карты, реклама, агрегатор;
- посадочная страница;
- оффер;
- цена или диапазон, если указаны;
- доставка;
- монтаж;
- рассрочка;
- гарантии;
- отзывы и рейтинг;
- фото и визуальные доказательства;
- товары или услуги в карточке;
- сильные элементы;
- слабые элементы;
- что применимо для проекта.

## Сравнение посадочных страниц

По посадочным проверяется:

- соответствие запросу;
- наличие цены или понятного способа узнать цену;
- ассортимент;
- сравнение моделей;
- доставка;
- монтаж;
- гарантии;
- отзывы;
- фото и видео;
- FAQ;
- региональность;
- CTA;
- мобильная версия;
- скорость понимания оффера;
- schema и базовая техническая корректность, если проверяется.

## Сравнение карточек Яндекс Бизнеса

По карточкам проверяется:

- рейтинг;
- количество отзывов;
- свежесть отзывов;
- ответы на отзывы;
- фото;
- товары и услуги;
- акции;
- адреса;
- телефоны;
- сайт;
- маршруты;
- заполненность профиля;
- признаки активности;
- соответствие карточки региональной посадочной.

Карточка конкурента не сравнивается с сайтом проект напрямую. Сравнивать нужно карточку с карточкой, посадочную с посадочной, оффер с оффером.

## Связка с Wordstat

Wordstat показывает спрос, конкурентный анализ показывает, кто этот спрос закрывает.

Для каждого кластера нужно определить:

- есть ли спрос;
- кто виден в поиске;
- кто виден в Картах;
- есть ли у конкурентов отдельная посадочная;
- есть ли у проекта аналогичная посадочная;
- индексируется ли посадочная проекта;
- есть ли трафик и цели в Метрике;
- нужна ли SEO-страница, статья, рекламная группа или доработка карточки.

## Связка с рекламой

Если конкурент виден в рекламе, нужно проверить:

- какой оффер используется;
- на какую страницу ведет объявление;
- закрывает ли проект этот же спрос;
- есть ли у проекта технически готовая посадочная;
- есть ли смысл тестировать этот кластер в Директе;
- может ли этот спрос позже закрываться SEO/GEO.

## Статусы конкурентных выводов

Каждый конкурентный вывод получает статус:

- `применимо`;
- `применимо после технических исправлений`;
- `требует проверки спроса`;
- `требует проверки конверсии`;
- `не применимо`;
- `наблюдение без действия`.

Нельзя переносить элемент конкурента в план проект только потому, что он есть у конкурента.

## Выходные артефакты

Результат конкурентного анализа:

- конкурентная карта по кластерам спроса;
- список решений для сайта;
- список решений для карточек Яндекс Бизнеса;
- список решений для рекламных объявлений;
- список тем для SEO-страниц и статей;
- вопросы агентству;
- обновление матрицы доказательств;
- обновление карты потерь и возможностей;
- корректировка приоритетов.

## Запреты

Нельзя писать:

```text
У конкурентов лучше сайт, нужно сделать так же.
```

Нужно писать:

```text
По кластеру "теплица цена Пермь" конкурент использует посадочную с ценами, доставкой и отзывами. У проект соответствующая региональная посадочная требует проверки в Вебмастере и Метрике. Решение применимо только после подтверждения технической готовности страницы.
```

## Проверка качества

Перед использованием конкурентного вывода нужно проверить:

- запрос;
- регион;
- дату проверки;
- тип выдачи;
- источник;
- применимость к проекту;
- не является ли вывод субъективной оценкой;
- есть ли действие, которое можно проверить после внедрения.
