# Регламент конкурентного анализа

Статус: **черновик-заготовка**. Заполняется после первой полной работы по конкурентному анализу. До этого — список областей, которые нужно закрыть, и предполагаемая структура документа.

## Назначение

Конкурентный анализ нужен не для описания рынка, а для выработки конкретных решений по сайту, карточкам, рекламе и контенту проекта. Регламент описывает, как собирать данные о конкурентах доказуемо и какие выводы из этого могут переходить в работы.

## Принципы

- Данные о конкурентах фиксируются по факту наблюдения, а не по ощущениям.
- Анализ ведётся по осмысленным сегментам (регионы, категории, тип запроса), а не общим списком "конкурентов".
- Каждое наблюдение превращается в строку матрицы доказательств с категорией `конкуренты` и конкретным выводом.
- Read-only правило: данные конкурентов собираются без взаимодействия с их сайтами и рекламой за пределами обычного просмотра.

## Источники

### Поиск и позиции

- Яндекс поиск по ключевым коммерческим и информационным запросам (ручной или через сервис мониторинга).
- Яндекс.Карты для локальных запросов.
- Google (опционально — Google даёт значимую долю органики: см. baseline Метрики проекта).

### Сайты конкурентов

- Публичные страницы категорий, карточек, статей, trust-pages.
- Robots.txt, sitemap.
- Проверка canonical, schema, meta.
- Structured snapshot через архив или screenshot-набор.

### Карточки Яндекс Бизнеса

- Количество отзывов, рейтинг.
- Полнота карточки: фото, услуги, цены, ответы на отзывы.
- Поисковые фразы, по которым появляется карточка (если доступны).

### Рекламные объявления

- Объявления конкурентов в поиске Яндекса по целевым фразам.
- Посадочные страницы рекламы конкурентов.
- Офферы, CTA, УТП.

## Минимальная матрица сегментов

Сегменты, по которым проводится анализ:

- Категория товара:
  - теплицы (арочные, двускатные, каплевидные, прямостенные, стеклянные);
  - парники;
  - беседки;
  - поликарбонат;
  - грядки;
  - комплектующие.
- Регион:
  - Пермь;
  - Екатеринбург;
  - Тюмень;
  - Ижевск.
- Тип запроса:
  - брендовые;
  - коммерческие высокочастотные;
  - коммерческие локальные;
  - информационные (как выбрать, сравнения);
  - B2B / промышленные.
- Канал видимости:
  - органический поиск;
  - платный поиск;
  - Яндекс.Карты;
  - агрегаторы и каталоги.

## Что смотреть у каждого конкурента

Для каждого значимого конкурента собрать:

### Поисковая видимость

- По каким группам запросов видим.
- По каким регионам видим.
- Какие типы страниц в топе (категория, карточка, лендинг, статья).
- Есть ли региональные поддомены или подпапки.

### Сайт

- Карточка товара: цена, характеристики, комплектации, варианты, FAQ, блоки доверия.
- Категория: сравнение моделей, навигация, фильтры.
- Trust pages: О компании, Доставка, Оплата, Отзывы, Портфолио.
- Формы захвата лидов: тип, длина, удобство.
- Мобильная версия: видна ли главная информация, доступны ли формы.
- Время загрузки.
- Schema разметка.

### Рекламные посадочные

- Если конкурент ведёт Директ на отдельные лендинги — какие.
- Офферы, CTA, цены, акции.
- Использование рассрочки, гарантии, монтажа как УТП.
- Наличие калькуляторов, конфигураторов.

### Карточка Яндекс Бизнеса

- Рейтинг, количество отзывов.
- Есть ли ответы на отзывы.
- Фото, услуги, цены.
- Связка с сайтом (переход, звонок).

### Контент

- Блог, статьи: темы, глубина, формат.
- Кейсы монтажа, видео.
- Сравнения моделей, guides.

## Формат вывода

Документ `<project> — Конкурентный анализ YYYY-MM-DD.md` содержит:

- Список конкурентов с короткой характеристикой.
- Таблицу разрывов (`gap analysis`):
  - колонки — конкуренты;
  - строки — элементы (оффер, карточка, trust pages, формы, schema, карточка Яндекс Бизнеса);
  - ячейки — краткое описание.
- Список решений, которые стоит применить к проекту, с приоритетами.
- Открытые вопросы.

Каждое решение уходит в матрицу доказательств с указанием источника (наблюдение у конкретного конкурента).

## Частота

- Полный конкурентный анализ — перед каждым крупным этапом работ (техфиксы, контент-масштабирование, рекламный рост).
- Обновление ключевых срезов — раз в 2-3 месяца.
- Оперативная проверка отдельных конкурентов — при появлении значимых сигналов (новая реклама, резкое изменение позиций).

## Что НЕ является конкурентным анализом

- Просто список сайтов без выводов.
- Сравнение цен без сегментации.
- Копирование оффера без проверки, работает ли он у конкурента.
- Выводы "у них так — сделайте нам так же" без связки с данными проект (возможно, у проекта этот сегмент не работает не из-за оффера).

## Что делать после первой работы

После первого полного цикла:
1. Снять статус `draft` и перевести в `final`.
2. Добавить реальные примеры конкурентов и методов сбора данных.
3. Зафиксировать инструменты, которые оказались полезными.
4. Дополнить типовые паттерны офферов и оформления, найденные у конкурентов.
