# Словарь метрик и сущностей

## Назначение

Документ фиксирует единый язык проекта.

Он нужен, чтобы бизнес, аналитик, агентство и разработчик одинаково понимали ключевые термины при чтении отчётов, ТЗ и вопросов агентству.

**Вход:** начало работы с документами фреймворка — отчётом, ТЗ, [[Матрица доказательств|матрицей доказательств]] или [[Модель приоритизации задач|моделью приоритизации]].  
**Выход:** согласованные определения, используемые без уточнений во всех документах проекта.  
**Gate:** словарь — обязательный артефакт Gate 0 (принятие словаря = условие готовности системы к работе). Проверяется также на Gate 3 (перед матрицей доказательств) и Gate 4 (перед адресными документами). См. [[QA Gate контроль качества]].

## Применение и обновление

- Открывать перед написанием отчёта, ТЗ или вопросов агентству — если термин может быть истолкован по-разному.
- Термин добавляется в словарь, если он используется минимум в двух документах фреймворка и его значение неочевидно.
- Общеизвестные понятия (клик, страница, бюджет) в словарь не вносятся.
- Решение о добавлении принимает Pavel + Codex + Claude; фиксируется в [[Учет времени, решений и действий]].

## Статусы доказанности

| Термин | Значение |
|---|---|
| Факт | проверенное наблюдение с источником и периодом |
| Гипотеза | предположение в формате: «Если [X] → то [Y] → проверим через [окно]» |
| Подтверждено | вывод проверен по заданному критерию в контрольном окне |
| Не подтвердилось | ожидаемая связь или эффект не найден |
| Недостаточно данных | данных не хватает для вывода |
| Невозможно отделить влияние | результат есть, но нельзя точно установить причину из-за параллельных факторов |

Статус присваивает Pavel + Codex + Claude. Критерии перехода между статусами — в [[QA Gate контроль качества]].

## Сущности фреймворка

| Термин | Рабочее определение |
|---|---|
| Этап | отдельный шаг флоу (Этапы 0–13); имеет вход, выход и gate-условие |
| Пакет | набор документов, создаваемых на группе этапов (Пакеты 0–6) |
| Gate | контрольная точка перехода между этапами; определяет условия, при которых можно двигаться дальше |
| Матрица доказательств | сводный документ, связывающий каждый вывод с источником, статусом доказанности и бизнес-смыслом |
| Карта влияния | документ, оценивающий измеренный эффект внедрённых изменений с учётом уверенности и ограничений |
| Baseline | зафиксированное состояние метрик до внедрения изменения; как правило, период 30 дней до даты изменения по данным Метрики или Вебмастера |
| Контрольное окно | период после изменения, за который проверяется эффект; длина зависит от типа изменения (SEO — 60–90 дней, технические правки — 14–30 дней) |

## Система трассировки

| Код | Название | Назначение |
|---|---|---|
| HYP | Hypothesis | гипотеза перед действием: «Если [X] → то [Y] → проверим через [окно]» |
| DEC | Decision | управленческое решение с обоснованием |
| ACT | Action | выполненное действие |
| CHG | Change | зафиксированное изменение в системе или на сайте |
| IMP | Impact | измеренный результат изменения |
| ACH | Achievement | достигнутая веха или цель |
| FLOW | Flow version | версия процессной карты; фиксирует, по какой редакции [[Сквозной флоу управляемой системы роста]] выполнялся цикл |

Хранятся в папке `_decisions/` проекта. Именование: `HYP-001-название.md`, `DEC-002-название.md`, `FLOW-001-название.md` и т.д.

## Воронка

| Термин | Рабочее определение |
|---|---|
| Показ | технический факт загрузки сайта, карточки или объявления в браузере пользователя |
| Клик | переход из источника на сайт или в карточку |
| Визит | сессия пользователя на сайте; в Яндекс Метрике новый визит начинается после 30 минут бездействия |
| Посадочная страница | первая или целевая страница, на которую приходит пользователь |
| Микроконверсия | действие, показывающее интерес, но не равное лиду; примеры: клик по номеру телефона, скролл до блока CTA, просмотр 3+ страниц |
| Коммерческое действие | звонок, форма, маршрут, переход в мессенджер — любое действие с коммерческим намерением; является микроконверсией или лидом в зависимости от результата |
| Лид | обращение с коммерческим намерением, переданное в обработку; отличие от коммерческого действия — факт передачи в CRM или менеджеру |
| Качественный лид | лид, соответствующий критериям бизнеса (география, товарная группа, бюджет запроса, без дублей); критерии уточняются в [[Цели и конверсии]] |
| Продажа | подтверждённая сделка или оплата |

## Каналы

| Термин | Рабочее определение |
|---|---|
| SEO | работа по увеличению органического трафика из поисковых систем |
| GEO | канал региональной видимости: геозависимые поисковые запросы, карточки на картах, локальная выдача |
| Яндекс Бизнес | инструмент управления карточкой организации в сервисах Яндекса (Карты, Поиск, Справочник); основной инструмент GEO-канала |
| Директ | платный трафик из Яндекс Директа |
| SERP | страница результатов поиска (Search Engine Results Page) |
| Wordstat | инструмент оценки рыночного спроса и сезонности по поисковым запросам; показывает не трафик сайта, а частотность запросов по рынку в целом |
| Вебвизор | инструмент анализа поведения пользователей по записанным сессиям в Яндекс Метрике |

## Технические сущности

| Термин | Рабочее определение |
|---|---|
| Sitemap | файл или набор файлов, через которые сайт сообщает поисковику URL для обхода и индексации |
| Canonical | указание основной версии страницы; *пример: если страница доступна по нескольким URL, canonical указывает, какой из них должен попасть в индекс* |
| 404 | страница не найдена |
| 5xx | серверная ошибка |
| `LOW_QUALITY` | сигнал Яндекс Вебмастера о низком качестве или недостаточной ценности страницы; *пример: страница-заглушка, страница с дублированным или очень коротким контентом* |
| `NOT_CANONICAL` | URL исключён из индекса как неканонический |
| Sampling / `sample_share` | доля данных, по которой построен отчёт в Метрике; значение ∈ [0; 1]; при `sampled: true` и `sample_share < 0.5` выгрузка считается непригодной, см. [[Выгрузка и качество данных]] |
| `data_lag` | задержка дозаписи данных: поведенческие факторы в Метрике досчитываются до 8 дней, расходы Директа подтягиваются до 21 дня, e-commerce — до 30 дней |
| Attribution | модель определения источника визита/конверсии; в Метрике по умолчанию `lastsign` (последний значимый источник); другие значения: `last`, `first`, `cross_device_last_significant` |
| Chunk / чанкинг | разбиение запроса к API на части из-за лимитов на количество метрик/строк; например, goal-метрики разбиваются по 200 ID на чанк |
| `disclosure` | ограничения Яндекса на раскрытие части строк в отчёте (обычно по конфиденциальности); проверяется при выгрузке |
| `contains_sensitive_data` | флаг наличия персональных данных (clientID, IP) в выгрузке; при `true` требуется обезличивание перед сохранением в vault |
| Goal-метрика | параметр API Метрики для запроса данных по конкретной цели счётчика; лимит — 500 goal-метрик на запрос, поэтому применяется чанкинг |

## Приоритеты

| Термин | Рабочее определение |
|---|---|
| P0 | блокер: останавливает индексацию, измерение, рекламу или коммерческие действия; требует устранения вне очереди; *пример: счётчик Метрики не работает, форма заявки сломана* |
| P1 | важная задача, влияющая на рост трафика, качество конверсий или управляемость проекта; *пример: посадочная не соответствует поисковому запросу, UTM-метки настроены неверно* |
| P2 | улучшение, полезное но не блокирующее текущий цикл; *пример: оптимизация мета-тегов на низкочастотных страницах* |

## Приоритизация гипотез

| Термин | Формула | Применение |
|---|---|---|
| ICE | Impact × Confidence × Ease | быстрая оценка гипотез; каждый компонент 1–10, итог = произведение |
| RICE | Reach × Impact × Confidence / Effort | расширение ICE с учётом охвата (Reach) и трудозатрат (Effort) |

Используется в [[Модель приоритизации задач]]. Impact — потенциальный эффект на метрики, Confidence — уверенность в результате на основе данных, Ease / Effort — сложность реализации.

## Деньги и эффективность

| Термин | Рабочее определение |
|---|---|
| CPL | стоимость лида (расходы / количество лидов) |
| CPA | стоимость целевого действия; в контексте рекламы — любое целевое действие, в бизнес-отчёте — стоимость продажи |
| CPC | стоимость клика |
| CTR | кликабельность (клики / показы × 100%) |
| ROMI | возврат маркетинговых инвестиций; рассчитывается только при наличии данных по выручке или марже; при отсутствии этих данных используется CPL как основная метрика эффективности |
| Стоимость ошибки | оценка потерь от неверного действия или технической проблемы; считается как: упущенный трафик × конверсия × средний чек |

## Правило

При использовании термина в значении, отличающемся от этого словаря:

1. Зафиксируй локальное определение прямо в документе с пометкой `[локальное определение]`.
2. Создай задачу на обновление словаря в [[Учет времени, решений и действий]].
