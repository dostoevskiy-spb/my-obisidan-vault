# RACI ролей и ответственности

## Назначение

Документ фиксирует, кто за что отвечает в проекте, и является матрицей доступов к инструментам — кто владеет Метрикой, Вебмастером, Директом, CRM и другими каналами.

RACI нужен, чтобы задачи не зависали между бизнесом, агентством, разработчиком и исполнителями.

**Вход:** вопросы к бизнесу и агентству из раздела «Порядок заполнения» закрыты, роли известны.  
**Выход:** заполненная матрица, факт согласования зафиксирован в [[Учет времени, решений и действий]].

## Обозначения

| Роль | Значение |
|---|---|
| R — Responsible | делает работу |
| A — Accountable | отвечает за итоговое решение и принимает результат |
| C — Consulted | консультирует, дает данные или экспертное мнение |
| I — Informed | должен быть в курсе |

В одной строке должен быть один `A`. Несколько `R` допустимо. Если ответственного за итог нет, задача неуправляема.

## Роли проекта

| Роль | Кто может быть |
|---|---|
| Бизнес | владелец бизнеса или лицо, принимающее бизнес-решения |
| Pavel / Codex | координатор процесса + аналитик; выступает как R или C, не является единственным A по решениям, влияющим на деньги |
| SEO-агентство | SEO-план, семантика, статьи, рекомендации |
| Контекстолог / агентство Директа | кампании, медиаплан, объявления, ставки, UTM |
| Разработчик / Bitrix | технические правки сайта |
| Контент / редактор | статьи, тексты, карточки, правки контента |
| Менеджер продаж | качество заявок, CRM, статусы, причины отказов |
| Ответственный за Яндекс Бизнес | карточки, отзывы, фото, данные организаций |

## Порядок заполнения

Уточни с бизнесом и агентством до заполнения матрицы (см. [[Запрос доступов и данных для коллег]]).

**Для бизнеса:**

- кто принимает финальные решения по приоритетам и бюджету → строка «Приоритеты и бюджет»
- кто выдаёт доступы к данным → строка «Доступы и данные»
- кто подтверждает качество заявок → строка «CRM / заявки / продажи»
- кто отвечает за CRM/продажи/звонки → строка «CRM / заявки / продажи»
- кто принимает [[ТЗ для разработчика]] → строка «Технические правки сайта»
- кто подтверждает результат после внедрения → строка «Проверка внедрения»

**Для агентства:**

- кто отвечает за SEO → строки «SEO-статьи и контент», «Wordstat / спрос»
- кто отвечает за Директ и медиаплан → строка «Яндекс Директ»
- кто отвечает за семантику → строка «Wordstat / спрос»
- кто отвечает за тексты объявлений и UTM → строка «Яндекс Директ»
- кто отвечает за SEO-статьи → строка «SEO-статьи и контент»
- кто отвечает за отчётность и комментарии → строка «Отчёты бизнесу»

## Шаблон RACI

В ячейках указывай роль из раздела «Роли проекта» или имя конкретного человека. Один `A` на строку. Колонка `Access Level`: `ro` (только чтение) / `rw` (чтение + запись) / `admin` / `none`.

**Пример заполненной строки:**

| Блок работ | R — делает | A — отвечает за итог | C — консультирует | I — информируется | Access Level | Комментарий |
|---|---|---|---|---|---|---|
| Яндекс Метрика | Pavel / Codex | Бизнес | SEO-агентство | — | rw | цели, счётчики, Вебвизор |

**Матрица:**

| Блок работ | R — делает | A — отвечает за итог | C — консультирует | I — информируется | Access Level | Комментарий |
|---|---|---|---|---|---|---|
| Доступы и данные |  |  |  |  |  | кто выдаёт доступы и подтверждает полноту |
| Яндекс Вебмастер |  |  |  |  | ro | только чтение; write-действия только по отдельному решению |
| Яндекс Метрика |  |  |  |  |  | цели, счётчики, Вебвизор |
| Яндекс Бизнес |  |  |  |  |  | карточки, действия, регионы |
| Яндекс Директ |  |  |  |  |  | кампании, расходы, UTM, конверсии |
| CRM / заявки / продажи |  |  |  |  |  | качество лидов и связь с деньгами |
| Wordstat / спрос |  |  |  |  |  | семантика и сезонность |
| Конкурентный анализ |  |  |  |  |  | выдача, карты, офферы, посадочные |
| SEO-статьи и контент |  |  |  |  |  | нельзя запускать без спроса и роли в воронке |
| Технические правки сайта |  |  |  |  |  | P0/P1/P2, критерии приёмки |
| Проверка внедрения |  |  |  |  |  | кто подтверждает результат |
| Отчёты бизнесу |  |  |  |  |  | кто согласует финальные формулировки |
| Приоритеты и бюджет |  |  |  |  |  | кто принимает управленческое решение |

## Критерии готовности

RACI считается заполненным, когда:

- [ ] Каждая строка матрицы имеет ровно один `A`
- [ ] Все роли назначены конкретным людям или организациям
- [ ] Вопросы из раздела «Порядок заполнения» получили ответы
- [ ] Колонка `Access Level` заполнена для всех инструментальных строк
- [ ] Факт согласования с бизнесом зафиксирован в [[Учет времени, решений и действий]]

## Правила

- Без заполненного `A` задача не должна попадать в работу.
- Если агентство предлагает действие, у действия должен быть владелец проверки.
- Если разработчик сообщил «сделано», это не равно «проверено».
- Если решение влияет на деньги, `A` должен быть со стороны бизнеса.
- Яндекс Вебмастер — только `ro`. Write-действия требуют отдельного явного решения с указанием A.
- До выдачи [[ТЗ для разработчика]] или [[Вопросы агентству]] — матрица должна пройти критерии готовности выше.
