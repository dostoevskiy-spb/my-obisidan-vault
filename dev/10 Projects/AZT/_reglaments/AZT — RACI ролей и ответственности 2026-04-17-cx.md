# AZT — RACI ролей и ответственности

## Назначение

Документ фиксирует, кто за что отвечает в проекте AZT.

RACI нужен, чтобы задачи не зависали между собственником, агентством, разработчиком, аналитиком и исполнителями.

## Обозначения

| Роль | Значение |
|---|---|
| R — Responsible | делает работу |
| A — Accountable | отвечает за итоговое решение и принимает результат |
| C — Consulted | консультирует, дает данные или экспертное мнение |
| I — Informed | должен быть в курсе |

В одной строке должен быть один `A`. Если ответственного за итог нет, задача неуправляема.

## Роли проекта

| Роль | Кто может быть | Уточнить у собственника/агентства |
|---|---|---|
| Собственник | владелец бизнеса или лицо, принимающее бизнес-решения | да |
| Внутренний координатор | человек, который собирает решения и двигает процесс | да |
| Аналитик / Codex | сбор, проверка, сопоставление данных, документы | да |
| SEO-агентство | SEO-план, семантика, статьи, рекомендации | да |
| Контекстолог / агентство Директа | кампании, медиаплан, объявления, ставки, UTM | да |
| Разработчик / Bitrix | технические правки сайта | да |
| Контент / редактор | статьи, тексты, карточки, правки контента | да |
| Менеджер продаж | качество заявок, CRM, статусы, причины отказов | да |
| Ответственный за Яндекс Бизнес | карточки, отзывы, фото, данные организаций | да |

## Минимальные вопросы для заполнения

Для собственника:

- кто принимает финальные бизнес-решения по приоритетам и бюджету;
- кто дает доступы к данным;
- кто подтверждает, какие заявки считаются качественными;
- кто отвечает за CRM/продажи/звонки;
- кто принимает ТЗ разработчику;
- кто принимает результат после внедрения.

Для агентства:

- кто отвечает за SEO;
- кто отвечает за Директ;
- кто отвечает за медиаплан;
- кто отвечает за семантику;
- кто отвечает за тексты объявлений;
- кто отвечает за SEO-статьи;
- кто отвечает за UTM и посадочные;
- кто отвечает за отчетность и комментарии по выполнению.

## Правила

- Без заполненного `A` задача не должна попадать в работу.
- Если агентство предлагает действие, у действия должен быть владелец проверки.
- Если разработчик сообщил "сделано", это не равно `проверено`.
- Если решение влияет на деньги, `A` должен быть со стороны собственника или согласованного координатора.
