# AZT — Карта влияния изменений

## Назначение

Документ связывает решения и изменения с динамикой показателей.

Цель — не просто ответить "что сделали", а понять, как это повлияло на сайт, SEO, Директ, Яндекс Бизнес, конверсии и коммерческие действия.

## Принцип

Изменение показателя после действия не доказывает влияние автоматически.

Каждая оценка влияния должна учитывать:

- дату изменения;
- затронутый сегмент;
- baseline до изменения;
- окно сравнения после изменения;
- параллельные изменения;
- сезонность;
- изменение рекламного бюджета;
- технические сбои;
- ограничения источников данных.

```markdown

```

## Статусы уверенности

- `нет данных` — контроль еще невозможен;
- `гипотеза` — есть наблюдение, но данных недостаточно;
- `вероятно связано` — связь видна, но есть внешние факторы;
- `подтверждено` — изменение и результат связаны по заранее заданному критерию;
- `не подтвердилось` — ожидаемый эффект не появился;
- `невозможно отделить от других факторов` — было слишком много параллельных изменений.

## Минимальный набор метрик для оценки влияния

Для технических изменений:

- HTTP-статусы;
- индексация;
- исключения в Вебмастере;
- ошибки sitemap;
- canonical;
- мобильность;
- скорость и доступность.

Для SEO/GEO:

- показы;
- клики;
- средняя позиция;
- индексируемость;
- посадочные страницы;
- поисковые запросы;
- региональная видимость.

Для Директа:

- расходы;
- клики;
- CTR;
- CPC;
- конверсии;
- CPL/CPA при наличии данных;
- качество посадочной;
- UTM и цели.

Для сайта:

- визиты;
- посадочные страницы;
- отказы;
- глубина;
- время;
- цели;
- формы;
- клики по телефону;
- переходы в мессенджеры.

Для Яндекс Бизнеса:

- показы карточки;
- переходы на сайт;
- звонки;
- маршруты;
- действия в карточке;
- источники Яндекс Поиск, Карты, Навигатор.
