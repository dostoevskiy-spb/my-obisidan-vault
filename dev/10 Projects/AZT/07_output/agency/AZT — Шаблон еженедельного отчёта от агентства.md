# AZT — Шаблон еженедельного отчёта от агентства

> [!INFO] Назначение
> Структура еженедельного отчёта по PPC-кампаниям AZT, согласованная между собственником (Павел Ламзин) и агентством Maximus Media. Цель — единый формат, чтобы цифры совпадали с обеих сторон и обсуждение шло по содержанию, а не по методологии.
>
> Периодичность: базовый еженедельный отчёт (§0-§6 + §10-§11). Расширенный раз в 2-4 недели или при значимых изменениях (добавляются §7-§9, §12).

## §0. Шапка отчёта

- **Период отчёта:** `с ДД.ММ по ДД.ММ` (`N` дней)
- **Дата среза:** `ДД.ММ.ГГГГ ЧЧ:ММ` (когда снят отчёт — Яндекс дополняет данные задним числом до 21 дня)
- **Атрибуция Метрики:** `LastSign` / `LastSignDirectClickOrLastSignOther` / `Last` — **обязательно указывать**
- **Источник данных:** Direct API v5 + Metrika API + (если есть) ссылка на Data Studio
- **Счётчик Метрики:** `87177110` (`azt-teplica.ru`)
- **Ответственный со стороны агентства:** `ФИО, контакт`

## Методологические правила отчёта

1. **Конкретика вместо общих формулировок.** Вместо «работаем над РСЯ» — «добавили `100` минус-фраз в `707716223`, `25` площадок в `709150485`».
2. **Явная атрибуция** в шапке — без неё цифры конверсий не интерпретируются.
3. **Дата среза** — важна, потому что данные за последние `7-21` день дополняются. Без неё сверка невозможна.
4. **Разбивка конверсий по целям** — нельзя агрегировать форма + тел + спасибо в одну цифру без разбивки. Это разные качества лидов.
5. **Разделение типов оплаты** — CPA «за клики» и «за конверсии» суммируются только с оговоркой.
6. **Приложенный XLSX** (а не скриншот из мессенджера) — для проверяемости. Скриншот допустим как дополнение.
