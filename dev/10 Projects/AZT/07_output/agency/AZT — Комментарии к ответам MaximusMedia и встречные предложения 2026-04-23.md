# AZT — Комментарии к ответам MaximusMedia и встречные предложения

## Назначение

Документ — исходящий ответ MaximusMedia по результатам разбора их ответов на список вопросов от 20–22.04. Фиксирует:

- построчные комментарии к каждому пункту из [[AZT — Входной список вопросов с ответами агентства 2026-04-22-cx]];
- решения и уточнения по встречным вопросам агентства (ГЕО, объявления, план недели);
- список параллельных задач со стороны AZT.

К документу прикладываются: [[AZT — Доказательство методологии частотности в перевыгрузке 2026-04-22]] с полной выкладкой цифр и сырыми данными Wordstat, плюс пересобранные CSV с раздельной частотностью по регионам ([Пермь.csv](azt-semcore-corrected-2026-04-22/Пермь.csv), [Тюмень.csv](azt-semcore-corrected-2026-04-22/Тюмень.csv)).

## 1. По частотности и методологии

### 1.1. Где `!`-частотность

**Ответ MaximusMedia.** Колонка временно убрана из-за системных расхождений в предыдущей версии; вернётся после корректной перевыгрузки через Wordstat с оператором `!`.

**Комментарий.** Принято, ждём в следующей итерации. Просьба: снимать именно через Wordstat API с оператором `!` перед каждым значимым словом (`!купить !теплицу`), не через широкое соответствие. В нашем доказательстве методологии зафиксировано, что в прошлой версии значения `!W` массово расходились с реальной точной частотностью — поэтому контроль качества при перевыгрузке критичен.

### 1.2. Около 27 строк без частотности

**Ответ MaximusMedia.** Это фразы из отчёта «Поисковые запросы», в Wordstat отсутствуют.

**Комментарий.** Логично. Предложение на следующую итерацию: для таких фраз попробовать проставить точную частотность (`!фраза`) — часть из них в Wordstat всё же появляется при точном соответствии. Если ноль — оставить ноль, но проверка добавит прозрачности.

### 1.3. Источник данных

**Ответ MaximusMedia.** Базовая частотность Wordstat, не прогноз Директа и не медиаплановая модель.

**Комментарий.** Принято. Проверили дополнительно через Wordstat API по 3 общим фразам за 19.04 (зафиксировано в [[AZT — Полная сверка семядра с Wordstat 2026-04-19]]): РФ-частотность `купить теплицу` = 555 813, `купить теплицу цена` = 43 921, `купить усиленную теплицу` = 10 446. Цифры в новом файле в 27–74 раза меньше — это исключает методологию «РФ по Wordstat», что согласуется с ответом.

### 1.4. Первоначальное ядро vs новая выгрузка

**Ответ MaximusMedia.** Старое ядро было «примерным для расчёта медиаплана и бюджета», новая выгрузка собрана заново, единый источник, цифры сошлись на 90% с Wordstat.

**Комментарий.** По общим фразам без привязки к городу в тексте цифры в новой выгрузке действительно сходятся с Wordstat — но с суммой двух городов (`geo=[50, 55]`), а не с отдельными Пермью или Тюменью (подробнее — п. 1.5). Новая версия чище старой, но методологический вопрос остаётся.

### 1.5. «Частотность брали отдельно по каждому региону»

**Ответ MaximusMedia.** «Разделено по регионам в файле семантики, в логике, аналогичной медиаплану».

**Комментарий — требует сверки по фактам.** По нашим замерам в новой перевыгрузке раздельной частотности по Перми и Тюмени нет:

- **59 общих фраз в файлах Пермь.csv и Тюмень.csv имеют идентичную частотность до последней цифры.** Если бы частотность собиралась раздельно, для общих фраз цифры различались бы (разные города — разный спрос).
- **Для 8 из 10 проверенных фраз цифра в файле совпадает с `geo=[50, 55]`** — сумма двух городов. Пример: «купить теплицу» в файле = 8 793, Wordstat по `geo=[50, 55]` на 22.04 = 9 449 (расхождение 6%, в пределах дневных колебаний).
- **Для 2 из 10 фраз цифра совпадает с суммой `geo=[11108, 11176]`** — Пермский край + Тюменская область. Пример: «теплица 6 6 цена» в файле = 656, сумма края+области = 644 (разница 1.9%). Методология смешанная — в одной колонке часть фраз по сумме городов, часть по сумме регионов.
- **Реальные раздельные частотности** (собраны нами 22.04 через Wordstat API по 97 фразам Перми и 72 фразам Тюмени): «купить теплицу» Пермский край = 10 436, Тюмень-город = 3 687. Для города Тюмень цифра в файле MM (8 793) **завышена в 2.4 раза** относительно реального спроса в Тюмени.

Полная раскладка с таблицами, сырыми данными Wordstat и SHA-хешами — в приложении [[AZT — Доказательство методологии частотности в перевыгрузке 2026-04-22]].

**Что предлагаем.** Взять наши пересобранные CSV ([Пермь.csv](azt-semcore-corrected-2026-04-22/Пермь.csv), [Тюмень.csv](azt-semcore-corrected-2026-04-22/Тюмень.csv)) как рабочую основу для кампаний и медиаплана. В них частотность реально раздельная: Пермь собрана по `geo=[11108]` (Пермский край), Тюмень — по `geo=[55]` (город Тюмень). Формат файлов совпадает с вашим — 4 колонки: основная + расширенная из поисковых запросов.

### 1.6. Можно ли разделить по регионам

**Ответ MaximusMedia.** «Разделено по регионам в файле семантики».

**Комментарий.** См. п. 1.5 — по факту разделены только списки фраз (по словам «пермь»/«тюмень» в тексте), но не сами цифры частотности. Нужна пересборка. Плюс просьба: приложить частотности, по которым в медиаплане от 08.04 рассчитаны 10 851 показов Пермь и 6 851 Тюмень — чтобы у нас была общая проверяемая база.

## 4. По гео и настройкам кампаний

### 4.1. Гео-таргет vs частотность

**Ответ MaximusMedia.** «Не понятен вопрос».

**Комментарий — переформулируем.** В кабинете кампании по Перми (`700165888`, `709131003` и новые ЕПК) таргетируют `RegionIds: [11108, -11110, -20237, -110916]` = Пермский край минус Соликамск, Берёзники, Чайковский район. А частотность в файле Пермь.csv собрана по сумме городов `geo=[50, 55]` (см. п. 1.5). Гео-таргета реальной кампании и гео-методологии частотности не совпадают.

**Вопрос.** Можно в следующей итерации частотность для пермской кампании собирать по `geo=[11108]` (с теми же исключениями, что и в кампаниях)? Если есть причина оставить по двум городам — дайте знать.

### 4.2. Какой набор исключений правильный (Соликамск, Берёзники, Чайковский)

**Ответ MaximusMedia.** «Необходимо согласовать разногласие» + встречный вопрос бизнесу.

**Позиция AZT.** Соликамск (~90 тыс.), Берёзники (~140 тыс.), Чайковский район (~80 тыс.) — живые рынки, по логистике доставка туда отработана. **Предлагаем снять все три исключения и оставить чистый `[11108]` на всех кампаниях Перми (старых и новых ЕПК).** Если по какой-то территории у вас есть аналитическое обоснование исключать — пришлите данные, обсудим. Параллельно это выровняет гео между старыми классическими кампаниями и новыми ЕПК (сейчас они отличаются — см. п. 4.3).

### 4.3. Почему исключения по Перми отличаются между кампаниями

**Вопрос остаётся.** Старые кампании исключают три района, новые ЕПК — только Берёзники. По пункту 4.2 выше предлагаем унифицировать: снять все исключения, оставить `[11108]` везде.

### 4.4. Тюмень — только город или область

**Ответ MaximusMedia.** В старых РК и новых РК с оплатой за клики — только город Тюмень (ID 55). В МК с оплатой за конверсии — вся Тюменская область (ID 11176).

**Комментарий.** Логика разделения понятна. Но для корректного планирования медиаплана нужны две отдельные колонки частотности в семантике Тюмени:

- Колонка 1: `geo=[55]` — для поисковых кампаний и РСЯ с оплатой за клики.
- Колонка 2: `geo=[11176]` — для Мастеров Кампаний с оплатой за конверсии.

Сейчас в файле одна цифра, и она по сути не про Тюмень вообще (см. п. 1.5). Для планирования ни одного из сценариев она не годится.

## 7. Вопросы на обсуждение

### 7.1. Минус-фразы (`завод`, `отзывы`, `ремонт`, `демонтаж`, `домиком`, `мини`)

**По `завод` — критично для брендового трафика AZT.** Риск — резать запросы «завод теплиц», «александровский завод теплиц». Предлагаем:

- Убрать `завод` из общего списка.
- Заменить точечными исключениями по конкурентам: `-тосненский -коломна -липецкий -пенза -рязань -вологда -тула -новгород`.

**По `отзывы`, `ремонт`, `демонтаж`, `домиком`, `мини`.** В текущей формулировке режут тёплый трафик (замещающий спрос — около 18 тыс. показов/мес по РФ). Обоснование с цифрами Wordstat — в [[AZT — Правки к плану агентства (расширенная сверка) 2026-04-20]] §3.3. Предлагаем переоформить с операторами (`-!ремонт`, `-[ремонт стиральных]`, `-[мини юбка]` и т.д.) или снять и добавить защитные точечные минусы.

**По `прямостенная`.** Сначала проверка в интерфейсе Директа — там нужно смотреть фактическое поведение, Справка Директа говорит, что при совпадении минуса с ключевой фразой минус игнорируется, но для вариаций поведение может отличаться.

### 7.2. Недостающие кластеры (гео, замещающий спрос, инфо)

**Все три группы подтверждаем:**

- **Головные гео-запросы по Перми** — суммарно ~10 тыс. показов/мес по Перми. В первую очередь: `теплицы пермь` (6 872), `поликарбонат для теплицы пермь` (797), `теплицы в перми от производителя` (521), `теплицы пермь недорого` (344) и ещё ~10 фраз.
- **Замещающий спрос** — отдельной группой с собственными объявлениями: «замена старой теплицы», «замена поликарбоната», «новая теплица взамен сломанной». Посадочные — отдельный лендинг с оффером «меняем теплицу под ключ». Суммарно ~18 тыс. показов/мес по РФ, интент конверсионный.
- **Инфо-кластер** — отдельной кампанией с низкими ставками, целью на охват, посадочными-гайдами. Не смешивать с коммерческими ключами в одной группе — CR разный, метрики размоются.

### 7.3. Позиционирование объявлений

**Подтверждаем все четыре правки:**

1. Заменить «Найдёте дешевле — сделаем скидку» (в 3 из 9 объявлений) на нейтральное «Прямая цена от завода» или «Без наценки посредников». Текущая формулировка позиционирует AZT как перекупщика, а не завод-производитель.
2. Вынести «Полигаль Восток» в заголовки или текст объявлений. Сейчас — только в подписи креатива, теряется как якорь качества.
3. Перенести «Гарантия до 10 лет» в первый заголовок хотя бы части объявлений. Сейчас — во втором заголовке, триггер доверия не работает на полную.
4. Добавить отдельную брендовую группу с заголовком «Александровский Завод Теплиц — Официальный Сайт». Бренд AZT +31% г/г по Wordstat (март 2026: 244 показа против 186 в марте 2025), трафик дешёвый и высококонверсионный.

## 8. Встречные вопросы MaximusMedia — ответы AZT

### 8.2. Согласование ГЕО по Тюмени (вся область или только город)

**Решение AZT.** Оставить существующую логику: в поисковых РК и РСЯ с оплатой за клики — только город Тюмень (ID 55); в МК с оплатой за конверсии — вся Тюменская область (ID 11176). Но семантика и частотности должны быть раздельные для каждого сценария — см. п. 5.

### 8.3. План MaximusMedia на неделю

**По пункту «семантика через отчёт „Поисковые запросы“»** — поддерживаем.

**По пункту «дополнительные Мастера Кампаний с оплатой за конверсии (парники, конкуренты, околотематические запросы)»** — поддерживаем концептуально, но с учётом инцидента с МК 18 апреля (зафиксирован в [[AZT — Инцидент с рекламой Мастер Кампаний 2026-04-18]]) просим перед запуском согласовать по каждой МК:

- цель в Метрике, на которую будет обучаться стратегия;
- недельный бюджет и прогноз CPA;
- минус-слова;
- посадочная страница.

Хотим избежать повторения ситуации, когда МК крутится несколько дней с нулевой конверсией при полном расходе бюджета.
