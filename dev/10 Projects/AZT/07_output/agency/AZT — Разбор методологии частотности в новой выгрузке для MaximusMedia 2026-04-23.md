# AZT — Разбор методологии частотности в новой выгрузке (для MaximusMedia)

## О чём этот документ

В прошлом сообщении вы ответили на наш вопрос о методологии сбора частотности так: *«Частотность брали отдельно по каждому региону из Вордстат. Разделено по регионам в файле семантики Пермь/Тюмень в логике, аналогичной медиаплану»*.

Я пошёл проверить это на цифрах — прогнал фразы из присланных таблиц через Wordstat по разным гео-настройкам. Результат получился неожиданным: по факту частотности в таблицах собраны не совсем так, как мы думали. Скорее всего — это техническая особенность сбора, которую можно легко поправить в следующей итерации. Ниже разбор с цифрами, чтобы было понятно, что именно я вижу, и чтобы нам с вами говорить на одном языке при пересборе.

Документ построен по фактам, все сырые данные Wordstat у меня сохранены — могу прислать архивом по запросу, чтобы любой шаг можно было воспроизвести.

## Короткая версия

1. В таблицах Пермь и Тюмень **59 общих фраз** (те, у которых нет названия города в тексте). **У всех 59 частотность совпадает до последней цифры** — как в таблице Пермь, так и в таблице Тюмень.
2. Проверка через Wordstat показывает, что цифры в таблицах близки к сумме двух городов `geo=[50, 55]` (Пермь-город + Тюмень-город), а не к отдельным регионам: для 8 из 10 проверенных фраз расхождение с `geo=[50, 55]` укладывается в дневные колебания Wordstat.
3. Для 2 из 10 фраз цифра совпадает не с суммой городов, а с суммой `geo=[11108, 11176]` (Пермский край + Тюменская область). То есть методология внутри колонки смешанная — часть фраз посчитана по городам, часть по регионам.
4. По реальному таргету активных кампаний (Пермский край `[11108]` для пермских кампаний, город Тюмень `[55]` для тюменских) цифры получаются другие. Особенно заметно по Тюмени — там частотность в таблице **завышена в 2–40 раз** (то есть в 2–40 раз больше реального спроса) относительно города. Пример: `теплица капелька купить` в таблице 286, реально в Тюмени 7.
5. Параллельно пересобрал частотность раздельно по правильному гео для всех фраз. Формат тот же, что у вас — можно брать как рабочую основу или использовать для сверки.

## Гео-запросы, использованные в проверке

Вот какие гео прогонял и зачем:

| Гео | GeoID | Что это | Где применялось |
|---|---|---|---|
| `geo=[50]` | 50 | Город Пермь (~1 млн населения) | Проверка, сходится ли цифра в таблице с одним городом |
| `geo=[55]` | 55 | Город Тюмень (~850 тыс.) | То же + правильная выгрузка по Тюмени |
| `geo=[50, 55]` | 50+55 | Сумма двух городов | Проверка гипотезы «сумма городов» |
| `geo=[11108]` | 11108 | Пермский край (~2.5 млн) | Правильная выгрузка по Перми — по гео активных кампаний |
| `geo=[11176]` | 11176 | Тюменская область (~1.6 млн) | Не использовалось отдельно |
| `geo=[11108, 11176]` | 11108+11176 | Сумма края и области | Проверка гипотезы «края+области» |

**Почему именно такие гео.** Я посмотрел через API, какой регион реально таргетируют активные кампании в кабинете:

- Пермские кампании (`700165888`, `709131003` и другие) — `RegionIds: [11108, -11110, -20237, -110916]`, то есть Пермский край минус Соликамск, Берёзники и Чайковский район. Верхний уровень = `geo=[11108]`.
- Тюменские кампании (`707716203`, `709131007` и другие) — `RegionIds: [55]`, только город Тюмень.

Поэтому «правильная» частотность для планирования — это `[11108]` по Перми и `[55]` по Тюмени. Для Мастеров Кампаний по Тюмени (с оплатой за конверсии), где вы используете область — это отдельно `[11176]`, тут нужна дополнительная колонка.
