# vault-digest

Обработка сырых логов сессий из `01 Inbox/sessions/raw/` — создание качественных саммари с wiki-links и связями.

## Workflow

### 1. Найти необработанные файлы

Прочитать все `.md` файлы в `01 Inbox/sessions/raw/` где frontmatter содержит `status: unprocessed`.

Если необработанных нет — сообщить "Нет новых логов для обработки" и завершить.

### 2. Для каждого raw-файла

#### 2a. Извлечь ключевую информацию
- **Заголовок**: из `# Сессия: ...`
- **Цель**: из `**Цель:** ...`
- **Дата**: из `**Дата:** ...` или frontmatter `session_date`
- **Проект**: из frontmatter `project`
- **Итоги**: секция `## Итоги` (если есть)
- **Лог**: записи `### HH:MM — ...` для таймлайна

Если нет секции `## Итоги` — использовать последние записи из `## Лог` как основу.

#### 2b. Найти связи в vault
Поиск связанных заметок:
- По проекту: `10 Projects/` — найти заметку проекта
- По тегам: grep по ключевым словам из итогов
- По wiki-links: проверить упоминания в существующих заметках
- По Areas: `20 Areas/Программирование/` и т.д.

#### 2c. Создать саммари-заметку

**Проверка дублей:** Поискать в `01 Inbox/` существующую заметку с тем же `source` в frontmatter. Если найдена — ОБНОВИТЬ её, не создавать новую.

**Имя файла:** `{project}--{session-date}--{slug}.md` (из заголовка сессии)  
**Папка:** `01 Inbox/`

**Формат:**

```yaml
---
type: session-log
project: {project}
source: {source из raw-файла}
session_date: YYYY-MM-DD
tags: [session-log, {project-tag}, ...]
created: YYYY-MM-DD
---
```

```markdown
# {Заголовок сессии}

**Проект:** [[{Project Name}]]
**Дата:** YYYY-MM-DD

## Цель
{Цель сессии}

## Результаты
{Ключевые пункты из Итогов, переформулированные для понимания вне контекста проекта}
- Что именно сделано
- Какие решения приняты
- Какие проблемы возникли и как решены

## Связи
- [[Project Name]]
- [[Related Note 1]]
- [[Related Note 2]]
```

#### 2d. Обновить raw-файл
Изменить frontmatter raw-файла: `status: processed`

#### 2e. Архивировать raw-файл
Переместить raw-файл из `01 Inbox/sessions/raw/` в `40 Archive/sessions/`

### 3. Обновить проекты (если есть)
Если проект существует в `10 Projects/` — добавить ссылку на session-log в заметку проекта.

### 4. Итоговый отчёт
Показать пользователю:
- Сколько сессий обработано
- Какие проекты затронуты
- Какие связи найдены
- Созданные заметки (со ссылками)

## Правила
- Никогда не создавать дубли — проверять `source` в frontmatter
- При обновлении существующей заметки — сохранять ручные правки пользователя
- Саммари должно быть понятно без контекста проекта
- Всегда добавлять wiki-links на связанные заметки
- Язык заметок — русский
- Для типа `session-plan-raw` — аналогичная обработка, но type саммари: `session-plan`
