Обработка сырых логов сессий — создание детальных саммари с архитектурой и wiki-links.
Эта команда выполняется НА СЕРВЕРЕ (или локально вручную).
Сбор raw-логов выполняется отдельно через /collect-sessions.

Используй skill vault-digest для обработки. Ключевые особенности:

1. Определи субмодуль каждой сессии (api, backend, front, etc.)
2. Прочитай CLAUDE.md проекта/субмодуля для контекста стека
3. Создай ДЕТАЛЬНЫЕ саммари: таблицы с полями, эндпоинты с methods, классы с namespace
4. Обязательно: бизнес-контекст (зачем), архитектурные решения, проблемы/решения
5. Маршрутизируй саммари в `10 Projects/{Project}/{submodule}/`
6. Обнови ADR при обнаружении архитектурных решений
7. Архивируй raw в `40 Archive/sessions/`
