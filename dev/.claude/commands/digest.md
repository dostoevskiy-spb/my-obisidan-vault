Обработка сырых логов сессий — создание саммари с wiki-links.
Эта команда выполняется НА СЕРВЕРЕ (или локально вручную).
Сбор raw-логов выполняется отдельно через /collect-sessions.

## Что делать

### 1. Найти необработанные файлы
Прочитай все `.md` файлы в `01 Inbox/sessions/raw/` где frontmatter содержит `status: unprocessed`.
Если таких нет — сообщи "Нет новых логов для обработки" и завершись.

### 2. Для каждого raw-файла

#### 2a. Извлечь ключевую информацию
- **Заголовок**: из `# Сессия: ...`
- **Цель**: из `**Цель:** ...`
- **Дата**: из frontmatter `session_date`
- **Проект**: из frontmatter `project`
- **Итоги**: секция `## Итоги` (если есть)
- **Лог**: записи `### HH:MM — ...`

Если нет `## Итоги` — используй последние записи из `## Лог`.

#### 2b. Найти связи в vault
- По проекту: ищи в `10 Projects/`
- По тегам и ключевым словам из итогов
- По существующим wiki-links в vault

#### 2c. Создать саммари-заметку
**Проверка дублей:** ищи в `01 Inbox/` заметку с тем же `source` в frontmatter. Если найдена — ОБНОВИ, не создавай новую.

**Имя файла:** `{project}--{session-date}--{slug}.md`
**Папка:** `01 Inbox/`

**Формат:**
```yaml
---
type: session-log
project: {project}
source: {source из raw}
session_date: YYYY-MM-DD
tags: [session-log, {project-tag}]
created: YYYY-MM-DD
---
```

```markdown
# {Заголовок сессии}

**Проект:** [[{Project Name}]]
**Дата:** YYYY-MM-DD

## Цель
{цель}

## Результаты
{ключевые пункты — кратко, понятно вне контекста проекта}

## Связи
- [[Project Name]]
- [[...]]
```

#### 2d. Архивировать raw
1. Обнови frontmatter raw-файла: `status: processed`
2. Перемести raw из `01 Inbox/sessions/raw/` в `40 Archive/sessions/`

### 3. Итоговый отчёт
- Сколько обработано
- Какие проекты затронуты
- Созданные/обновлённые саммари
- Найденные связи

## Правила
- Саммари должно быть понятно БЕЗ контекста проекта
- Всегда добавлять wiki-links
- Язык — русский
- Не создавать дубли (проверять source)
