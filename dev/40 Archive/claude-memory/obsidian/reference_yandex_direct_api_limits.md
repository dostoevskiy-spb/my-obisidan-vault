## МК-кампании («Мастер Кампаний») невидимы через `campaigns.get` v5

**Проблема:** МК-кампании в интерфейсе Директа отображаются, но через API v5 `campaigns.get` не возвращаются ни по какому из типов:
- `SMART_CAMPAIGN` → 0 результатов
- `UNIFIED_CAMPAIGN` → 0 результатов
- `MCBANNER_CAMPAIGN` → ошибка 8000 (тип не поддерживается)
- Запрос по конкретному `Id` МК → пустой массив

**Пример (AZT кабинет 2026-04-20):** в интерфейсе видны 4 МК-кампании (`709149630`, `709150108`, `709179545`, `709179711`), все активны с расходом и показами. Через API — невидимы.

**Как работать:**
1. **Интерфейс** — главный источник для МК (скриншот от собственника/менеджера)
2. **`direct_report`** — МК попадают в статистические отчёты по CampaignId, там их можно агрегировать
3. **НЕ тратить units** на попытки вытащить МК через `campaigns.get` с разными типами

## `changes.check` / `changes.checkCampaigns` не ловит создание новых МК

**Проблема:** аудит изменений за N часов через `direct_audit_changes_since` отдаёт пустой `changed_campaign_ids: []`, даже если за этот период агентство создало новые МК в кабинете.

**Следствие:** автоматический daily-audit пропустит создание МК. Надо либо:
- Ежедневно запрашивать скриншот списка кампаний у клиента/агентства
- Сравнивать `account_summary.campaigns_count` через снапшоты (но при создании МК в отдельном разделе кабинета счётчик может не меняться)

## Ошибка 4001 в `account_summary` при `>1000` кампаний

`account_summary` передаёт все CampaignIds одним массивом во вложенные вызовы. На больших кабинетах (`>1000` кампаний, включая архив) API возвращает ошибку «Превышено допустимое количество идентификаторов в массиве SelectionCriteria.CampaignIds». Для таких кабинетов счётчики adgroups/ads/keywords будут с `error:...` в ответе — но `campaigns_count` корректный.

## `changes.check` ограничен 7 сутками назад

`Timestamp` в `changes.checkCampaigns` и `changes.check` не может быть старше `7 суток`. Для длинной истории нужно собирать снапшоты по расписанию (каждые 3-5 дней) и диффить вручную.

## Поддерживаемые типы кампаний в API v5

Валидные значения для `types` в `campaigns.get`:
- `TEXT_CAMPAIGN` — классическая текстовая
- `UNIFIED_CAMPAIGN` — ЕПК (Единая Перформанс-Кампания)
- `DYNAMIC_TEXT_CAMPAIGN`
- `MOBILE_APP_CAMPAIGN`
- `SMART_CAMPAIGN`
- `CPM_BANNER_CAMPAIGN`

Ошибка 8000 на других значениях. Для МК (Мастер Кампаний) отдельного типа в API v5 нет — см. выше.
